Using SSH to Secure WiFi

Say you are at McDonalds and you have your laptop with you to browse the Internet through the free Wi-Fi, it's a good idea to use an encryption tunnel to your home router using SSH (Secure Shell). If you'd like to also know about OpenSSH then I have that covered here. That way your traffic is routed through this tunnel directly from your laptop to your home router.

The router is conducting port forwarding by passing packets on to your home ssh server running linux. This tunnel is protected from Wi-Fi sniffers who would see nothing but encrypted data.

Now it won't matter how shoddy the Wi-Fi source is, McDonalds or even at an Airport for that matter, because your data is inside an encrypted tunnel.

Since you are running from a shell you can run the text browser lynx and the text email program mutt, thereby surfing like a nerd. Nice!

So in my example, I am using Kali linux as the ssh server, that means I will have shell access to the system. I have set it up to log in as a normal user and not the root account. Here's how with line 3 adding the user to the sudo group:

# useradd -m spoon
# passwd spoon
# usermod -a -G sudo spoon
# chsh -s /bin/bash spoon

The laptop is using the free Wi-Fi at McDonalds to access the internet. The connection goes to my home router. Therefore, the link from the Laptop in McDonalds, to the home router, is encrypted. To make this work I configured my home router to port forward ssh connections to an inside private ip address on my network. From there the connection goes back out through my home router to the internet unencrypted. At least I feel safer when using the free Wi-Fi because no one can eave's drop on my activity.
